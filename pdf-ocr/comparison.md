# DeepSeek-OCR vs PaddleOCR 对比测试

**测试时间：** 2026-03-18
**测试素材：** 《起卦秘籍》扫描版 PDF，取第 4、11、21 页
**图片分辨率：** 2x 渲染（原页面 × 2，约 1400px 宽）

## 版本信息

| 工具 | 版本 |
|------|------|
| PaddleOCR | 3.3.2（PP-OCRv5 server 模型） |
| deepseek-ocr | 3.3B 参数，F16 量化，via ollama（本地运行） |

## 速度对比

| 工具 | 速度 |
|------|------|
| PaddleOCR | 约 5~10s / 页（含模型初始化后） |
| deepseek-ocr | 约 12~22s / 页 |

## 识别质量对比（同一页面）

| 项目 | PaddleOCR | deepseek-ocr |
|------|-----------|--------------|
| 中文正文 | ✅ 准确 | ✅ 准确 |
| 专业术语（六爻） | ✅ **"六爻"**（正确） | ❌ **"六交"**（误识别） |
| 标点符号 | ✅ 分号、书名号完整 | ✅ 基本正确 |
| 头部噪声 | ✅ 无 | ❌ 有 `atts` 噪声行 |
| 省略号 | ✅ `……` | ✅ `……` |

## 结论

**对中文书籍扫描 OCR，PaddleOCR 优于 deepseek-ocr（本地 3.3B）：**

1. **速度更快**，吞吐量约 2x
2. **专业术语更准确**（"六爻"而非"六交"）
3. **无头部噪声**，不需要额外过滤
4. deepseek-ocr 的优势在于理解上下文，但 3.3B 量级对古文专词识别不如专用 OCR 模型

**最终方案：** PaddleOCR 负责识别，DeepSeek API（`deepseek-chat`）负责清洗页码/页眉噪声。
